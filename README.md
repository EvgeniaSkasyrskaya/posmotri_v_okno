## Проектная работа «Посмотри в окно»

### Цель работы:
дополнить CSS-стили для готового интерактивного приложения «Посмотри в окно», чтобы интерфейс соответствовал макету, сохранив существующую функциональность и минимально изменяя HTML-структуру

### Стек технологий
- HTML5 (готовая разметка с использованием <template>)
- CSS3 (Flexbox, Grid, кастомные формы, состояния элементов)
- JavaScript (готовый функционал приложения)
- Pixel Perfect подход
- Работа с частично готовым проектом

### Основные этапы

#### Подготовка
Изучение макета проекта  
Анализ готовой HTML-разметки и JavaScript-логики  
Понимание работы тегов <template> для динамического контента  

#### Создание базового лейаута
Настройка Flexbox для контейнера .page  
Расположение секций .result и .content__details  
Создание скроллируемой области для списка карточек  

#### Уточнение размеров элементов
Настройка размеров видео-контейнеров  
Расположение карточек с использованием gap  
Оптимизация изображений с помощью object-fit  

#### Стилизация текста
Оформление заголовков и акцентных элементов  
Настройка line-height и line-clamp для текстовых блоков  
Управление переполнением текста  

#### Кастомные элементы форм
Создание класса .visually-hidden для скрытия элементов  
Вёрстка кастомных чекбоксов с псевдоэлементами  
Стилизация текстового поля ввода  
Работа с состояниями :focus, :focus-visible, :hover  

#### Кнопки и их состояния
Создание общих стилей для кнопок  
Индивидуальная настройка кнопок «Найти» и «Показать ещё»  
Реализация состояний :hover, :active, :focus-visible  

#### Состояния карточек
Стилизация текущей выбранной карточки  
Настройка состояний при наведении и активности  
Работа с обводкой для фокусировки  

#### Улучшение UX
Добавление cursor: pointer для кликабельных элементов

#### Позиционирование прелоадеров
Настройка относительного позиционирования для контейнеров  
Корректное отображение анимаций загрузки  

### Ключевые особенности проекта
- Работа с готовым JavaScript-приложением без изменения его логики
- Использование динамических шаблонов <template>
- Создание полностью кастомных элементов форм
- Сохранение существующей функциональности при стилизации
- Точное соответствие Pixel Perfect макету
- Минимальное вмешательство в готовую HTML-разметку
